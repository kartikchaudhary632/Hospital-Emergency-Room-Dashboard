# 🏥 Hospital Emergency Room Dashboard

Hi! 👋

This is my Hospital Emergency Room Dashboard project, created using Microsoft Excel.

I built this project to practice and improve my data analytics skills by working on a complete project — starting from raw data and ending with an interactive dashboard.

The main idea behind this project was simple: take hospital emergency room data, clean it, analyze it, and present important information in a way that is easy to understand.

📌 About the Project

Hospital emergency rooms generate a lot of patient data every day, but raw data alone is difficult to understand.

In this project, I tried to answer questions like:

- How many patients visited the emergency room?
- What is the average waiting time?
- Are patients being attended to on time or facing delays?
- How satisfied are the patients?
- How many patients were admitted?
- Which age groups visit the emergency room most often?
- What is the gender distribution of patients?
- Which departments receive the most referrals?

To answer these questions, I created an interactive Excel dashboard where users can explore the data using Year and Month filters.

🛠️ Tools I Used

- Microsoft Excel
- Power Query
- Power Pivot
- DAX
- Pivot Tables
- Pivot Charts
- Excel Slicers
- Data Visualization

🔄 Project Process

I followed a complete data analytics workflow:

Raw Data → Data Cleaning → Data Transformation → Data Modeling → Analysis → Visualization → Interactive Dashboard

1️⃣ Understanding the Data

Before creating the dashboard, I explored the dataset and understood the available columns.

I identified important information required for the analysis, such as patient details, waiting time, admission status, satisfaction score, age, gender, and department referrals.

2️⃣ Cleaning and Transforming the Data

I used Power Query to prepare the dataset for analysis.

During this stage, I:

- Checked column data types
- Cleaned and transformed the data
- Removed unnecessary information
- Fixed data inconsistencies
- Prepared the data for analysis
- Created a Calendar Table

This process helped me understand the importance of clean and structured data before starting any analysis.

3️⃣ Data Modeling

After cleaning the data, I used Power Pivot to build the data model and manage relationships between tables.

I also created calculated columns and categories for better analysis, including patient age groups and patient waiting-time status.

The waiting-time status was divided into:

- On-Time
- Delay

These categories helped me analyze patient demographics and emergency room efficiency more clearly.

📊 Building the Dashboard

Once the data was ready, I started building the dashboard using Pivot Tables, Pivot Charts, Slicers, and Excel formatting.

The dashboard includes analysis of:

- Total number of patients
- Average waiting time
- Patient satisfaction
- Admission status
- On-Time vs Delayed patients
- Gender distribution
- Patient age groups
- Department referrals

I also added Year and Month slicers so users can interact with the dashboard and explore specific time periods.

🎨 Dashboard Design

I wanted the dashboard to be useful as well as visually attractive.

While designing it, I worked on:

- Dashboard layout planning
- KPI card design
- Custom slicer formatting
- Chart formatting
- Consistent visual theme
- Removing unnecessary gridlines
- Sheet navigation using hyperlinks
- Interactive filters
- Easy-to-read visualizations

One of the main challenges was presenting a large amount of information on one screen without making the dashboard difficult to understand.

💡 What I Learned

This project was a great learning experience for me.

While working on it, I improved my understanding of:

- How to approach a data analytics project from the beginning
- How to clean datasets using Power Query
- How data modeling works in Excel
- How to use DAX for calculated columns

- How to select suitable charts for different types of data
- How to build interactive dashboards

Most importantly, I learned that creating a dashboard is not just about making charts. The real goal is to turn data into information that people can easily understand and use.
